# Getting started

New to Hoptimator? Read these in order:

1. **[Quickstart](quickstart.md)** — five-minute walkthrough on a local
   Kubernetes cluster. Build, deploy the demo, create a materialized view,
   watch it become a real pipeline.
2. **[Concepts](concepts.md)** — the vocabulary the rest of the docs assume:
   Database, View, Pipeline, Engine, Deployer, Template, Subscription, Hint.
3. **[Architecture overview](architecture.md)** — the path from a SQL
   statement to running infrastructure, and a map of the modules in the repo.

If you'd rather start with the high-level pitch, the
[engineering blog posts](../resources/learn-more.md) cover the design ideas and
LinkedIn's production use cases.
